Users Guide

Creating and connecting devices

To create a device, middle click, and then click on the name of the appropriate device group. A list of the devices in that group will then be displayed. Clicking on the name of the device will then create a new instance of that device. For example, choosing Input devices, then MousePad will create a new mouse input device. The devices are described in SigilDevices.

Devices can be connected by shift-dragging from the device producing the output to the device to receive the input. The connection can be removed by ctrl-clicking on the arrow on the interconnecting line. The width of the signal path is shown by the number of lines connecting the devices; a gray line indicates that only objects pass through the connection.

Selection

Selections can be made by either dragging the mouse to make a new selection or by shift-clicking to add a device to the current selection. Selected devices can be grouped by then right-clicking and choosing “Group” from the popup menu. Groups can be expanded by right-clicking on the group and choosing “Expand”, similarly bringing up the context menu for a device in a group will bring up a “Contract” option.

Saving and loading

Setups can be saved and loaded as desired; all device properties will be maintained. The save and load options can be accessed via the global popup menu (brought up by right-clicking over an empty area), or from the bar at the bottom of the screen. The bar will only appear when the mouse pointer is near the bottom of the window to maximize use of screen space.

Controls

The following keys and controls are used in SIGIL:

  • Double-click while over a device brings up the device interface.

  • Shift-drag while over a device drags a connection to another device.

  • Shift-click adds a device to the current selection.

  • Click, while not over a device deselects.

  • Drag drags a selection.

  • Ctrl-click, if over an arrow connecting two devices, deletes the connection

  • Right-click brings up the context menu for the device if over a device, the menu for a selection if there is a selection, and the system menu if not over a device and there is no selection.

  • Middle-click brings up the device selector.

All of the following commands are accessible via the menus, but also have associated keyboard shortcuts.

  • Alt-S saves the current setup.

  • Alt-O opens a setup.

  • Alt-D brings up the device selector

  • Alt-P updates a device (loads a new class file).

  • Alt-E evolves a device, if the device supports this.

  • Alt-A brings up the about box.

  • Alt-Z undoes the last action.

  • Alt-R redoes the last undo.

  • Alt-Y synchronizes all devices

  • Alt-U desynchronizes all devices

Menus

Global

The global menu is displayed when right clicking over an empty area of the screen, when there is no selection. The following commands are available from the system menu:

  • Save displays a filechooser from which the current setup can be saved. The user will be prompted for an annotation to add or edit, which will be saved to the file. The version number for a setup will be incremented every time it is saved. Files are saved with a “.sgc” extension.

  • Load displays a fisheye filechooser, from which SIGIL files can be selected. A loaded setup will replace the current configuration. Note that the meta information about the file is displayed in the chooser, beside the filename, when sufficient space is available. The last modified date, the version number and the annotation are all displayed.

  • New clears the current setup, after warning the user.

  • Merge is similar to load, but the devices in the new setup will be added to the current setup, rather than replacing them.

  • Synchronize all switches all devices into synchronized mode.

  • Desynchronize all desynchronizes all devices which support desynchronization. This provides lower

latency, but can lead to problems when multiple signal paths are combined.

  • Undo undoes the last undoable action. Undoable actions include load, merge, delete, or adding a device. The number of undoes available is shown beside the menu entry.

  • Redo redoes the last action undone. The number of redoes available is shown beside the menu entry.

  • Snapshot forces the current setup to be placed on the undo buffer. This is useful while experimenting with device parameters, in order to allow returning to a known working setup.

  • Undo enabled can be checked or unchecked. Unchecking it will clear the undo history.

  • Devices... brings up the device selector.

  • Update Device... brings up a dialog box asking for a device name. A device with that name will then be loaded from disk if possible, and then added to the device selector. This is useful if the module index file does not reflect the actual modules present.

  • About... brings up the about box.

Device context

Right-clicking on a device brings up the following menu:

  • Delete removes the device from the setup, disconnecting it as necessary.

  • Rename brings up a dialog to change the displayed name of the device.

  • Evolve... if shown, goes to the evolutionary interface to mutate the device.

  • If the device is in a group the following additional commands are available:

    • Ungroup removes this device from its group.

    • Contract causes the device and all of its siblings to contract back into the parent group.

  • If the device is a group the following additional commands are available:

    • Expand causes the group to expand, revealing all of the sub-components.

    • Break apart expands the group and then ungroups all of the components within.

    • All Synchronous can be checked or unchecked. It is checked if every member of the group is synchronous, unchecked otherwise. Checking it makes all members synchronous, unchecking it makes all members non-synchronous.

  • Synchronous shows the current synchronous status of the device. The state can be toggled by checking or unchecking.

  • Show source... brings up a window containing the source code for the device if it is available. The command will not be shown if the source cannot be found.

Selection

When there is a selection, right-clicking will bring up the following commands:

  • Group adds all components in the selection to a new group. A dialog box will be presented to request a new name for the group.

  • Delete will remove all the components in the selection.

Device selector

The device selector shows a list of each of the types of devices in SIGIL. Clicking on a type group will expand it, showing all of the actual devices in that group. Clicking on the group name again will cause that group to contract. When the list becomes larger than the available space, scroll arrows will appear at the top and bottom; clicking and holding on these will scroll the list. Clicking on a device will cause a new instance to be created. The device selector can be dragged around by grabbing any part of the device window which is not a group or device name.

Common problems

  • Signals seem to be getting lost

    • The buffer size can be adjusted via the “Buffers” option, from the context menu for a device. The default buffer size is usually more than enough though, so it is more likely that the signals are simply being delayed due to CPU overload.

  • Devices won’t connect

    • Check that the connection would not violate the topology constraints; that is no loops would exist in the device graph. Also check that the connection is not being made to a device that can only take a single signal, and which already has a connection. Some devices, like the graphing devices are “terminating” and do not produce output, and thus they have no output to connect to other devices. These devices have dark bars at the top and bottom to indicate this.

  • Signals out of sync when using Combiner, Switch, etc

    • Try resynchronizing the entire setup (Alt-Y or select “Synchronize all” from the system menu).

  • High latency

    • Latency can sometimes be improved by switching to semi-synchronous mode. Do this using Alt-U or “Desynchronize all” from the system menu, but be aware of the synchronization problem noted above.

  • Memory is low

    • Try disabling undo; if the setup is very complicated, undo can use a large amount of memory.

  • Cannot receive backwards passed objects

    • Devices that wish to receive backwards passed objects must actually be in the signal path; they cannot simply be attached as leaf nodes to devices higher up the device graph.

The filesystem structure

There are four main directories in the SIGIL file structure. If SIGIL is being run from a jar file, the current path is the root path of the jar file. The structure is:

  • . The current path, which must contain all of the device classes and any auxiliary files they require.

    • sigil contains all of the class files necessary for the main system.

    • duotonic holds the class files for the sound-related subsystem; speech, MIDI and the Shepard tone synthesizer. The classes “duotonic.PositionDemo”, “duotonic.Duotonic” and “duotonic.ParadoxSynthesizer” can all be executed to give demos of the some of the various sound functions.

    • lib contains all of the auxiliary files necessary for SIGIL and its modules to operate. This includes the English phonetic translation tables, the Markov model of the English language and the Earwig standard library (see UsingEarwig).

    • allophones contains samples of each of the phonemes used by the system, as spoken by the author. The file “phoneme.lst” in the “lib/” directory contains a complete list of these. Note that the filenames must be uppercase.

    • paradox contains short loops for each of the 12 Shepard tones, in N1.WAV...N12.WAV. Again, all names must be in uppercase.

  • A 

    sig·il  n. 

   1. A seal; a signet.
   2. A sign or an image considered magical.

  • Q Sample timing seems a bit unreliable.

  • A Unfortunately Java does not have very effective real-time capabilities. Timing is not accurate in Sigil.

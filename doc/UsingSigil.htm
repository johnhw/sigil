Using Sigil

Sigil is a tool for rapidly constructing sophisticated gesture recognition and sonification prototypes using a visual programming metaphor. It includes a large number of pre-built components for the manipulation and sonification of gesture. Sigil runs under Java and so is portable to many platforms.

Installing

Sigil runs on any environment supporting Java. Windows and Linux have been tested. Versions 1.3 and 1.4 of the Java Virtual Machine are known to work -- 1.5 should as well, but it's not been tested. If you don't have the Java runtime, get it at http://java.sun.com.

If you just want to run Sigil, download sigil.jar (1.2Mb). Run with java -jar sigil.jar

If you want to develop Sigil, download sigil.zip(4.3Mb), which includes the source code. Unzip to a directory of your choice. On Windows you can use run.bat. On other systems just execute java -classpath . sigil.SignalSystem. Some example setups can be found in the directory /setups.

Recompiling

Recompile with javac *.java sigil\*.java duotonic\*.java.

Troubleshooting

If the system fails to run, check the file errlog for details of the problem. If you receive errors about modules not being in the path, you can remove the offending modules from the file modules.txt in the /lib subdirectory.
